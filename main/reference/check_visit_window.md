# Check Visit Window Deviations

Check if completed visits were conducted within the specified visit
window. Receives planned visit data generated by
[`generate_planned_visit_dates`](https://insightsengineering.github.io/r.pkg.template/reference/generate_planned_visit_dates.md)
and checks if actual visit dates are within the window range (wp_start
to wp_end).

## Usage

``` r
check_visit_window(planned_dates, pdno = "8.4.1")
```

## Arguments

- planned_dates:

  Data frame from
  [`generate_planned_visit_dates`](https://insightsengineering.github.io/r.pkg.template/reference/generate_planned_visit_dates.md).
  Must contain: SUBJID, VISIT, VISITNUM, planned_date, wp_start, wp_end,
  wp_type, wp_value, actual_date, status, first_dose_date

- pdno:

  Character string specifying the protocol deviation number for this
  check (default: "8.4.1")

## Value

List with the following components:

- has_deviation:

  Logical. TRUE if there are out-of-window visits

- messages:

  Character vector. Deviation description messages

- details:

  Data frame. Out-of-window visit details with columns:

  - PDNO: Protocol deviation number specified by `pdno` parameter

  - SUBJID: Subject ID

  - first_dose_date: Subject's first dose date

  - VISIT: Visit name with deviation

  - VISITNUM: Visit number

  - planned_date: Planned visit date

  - actual_date: Actual visit date

  - wp_start: Window start date

  - wp_end: Window end date

  - wp_type: Window type

  - wp_value: Window value

  - deviation_days: Days deviated from planned (positive=late,
    negative=early)

  - total_completed_visits: Total completed visits for subject

  - out_of_window_count: Number of out-of-window visits

  - TBNAME: Table name from planned_dates, empty if not available

  - DESCRIPTION: Description of the deviation for each record

- planned_visits:

  Data frame. Complete input planned visit information

## Details

### Usage Workflow

This function is designed to work with `generate_planned_visit_dates`:

    # Step 1: Read visit schedule and generate planned visit dates
    visitcode <- read_visitcode_file("visit_schedule.xlsx")
    planned_dates <- generate_planned_visit_dates(data = study_data)

    # Step 2: Check visit window deviations
    result <- check_visit_window(
      planned_dates = planned_dates
    )

### Out-of-Window Criteria

For completed visits (status == "completed"):

- If actual visit date is within window (wp_start \<= actual_date \<=
  wp_end): Compliant

- If actual visit date is outside window (actual_date \< wp_start or \>
  wp_end): Deviation

### Window Types

Windows are calculated by `generate_planned_visit_dates` based on visit
schedule:

- +/-3d: 3 days before and after planned date

- +2d: Can only be delayed by 2 days

- -2d: Can only be early by 2 days

## Examples

``` r
if (FALSE) { # \dontrun{
# Load data and visit schedule
data <- read_raw_data("path/to/data")
visitcode <- read_visitcode_file("path/to/visitcode.xlsx")

# Generate planned visit dates (uses visitcode variable automatically)
planned_dates <- generate_planned_visit_dates(data = data)

# Check visit window deviations
result <- check_visit_window(planned_dates)
print(result)

# Access deviation details
if (result$has_deviation) {
  window_details <- result$details
  # Get subjects with out-of-window visits
  subjects_with_issues <- unique(window_details$SUBJID)
}
} # }
```
